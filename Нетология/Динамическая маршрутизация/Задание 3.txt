Задание 3. Оповещение соседей о потерянном маршруте
После установления соседства, отключите Loopback 0 на Router4 и посмотрите, каким образом произойдет оповещение соседей о потере маршрута. Посредством чего OSPF маршрутизатор оповещает соседей о потерянном маршруте?

Выполнение задания:
1. Отключение Loopback0 на Router4:
en
conf t
interface loopback
shutdown

2. Наблюдение за трафиком OSPF:
Посредством чего OSPF маршрутизатор оповещает соседей о потерянном маршруте?

OSPF маршрутизатор оповещает соседей о потере маршрута посредством отправки OSPF LSA (Link State Advertisement) пакетов. 
В данном случае, Router4 сгенерирует и разошлет LSA, в котором будет указано, что линк к Loopback0 больше не существует. 
Соседи Router4 (в данном случае R1) получат эту LSA, обновят свою базу данных о состоянии каналов (Link State Database - LSDB), 
пересчитают SPF-алгоритм (Shortest Path First algorithm), и обновят свою таблицу маршрутизации, удалив маршрут к Loopback0 на Router4.

Типы LSA пакетов:
1. Router LSA (Type 1): Router LSA информирует о состоянии линков маршрутизатора (включая стоимость) для всех маршрутизаторов в данной area. Когда интерфейс Loopback0 на Router4 отключен, информация об этом линке будет удалена из Router LSA.
2. Network LSA (Type 2): Network LSA описывает состояние линков в transit network (сеть, где есть несколько маршрутизаторов). В данной сети это не очень актуально, но если бы Loopback0 находился в transit network, Network LSA также обновилась бы.
3. Summary LSA (Type 3): Summary LSA используются для анонсирования сетей между areas. Поскольку все маршрутизаторы в area 0, Summary LSA не играют прямой роли в данном случае.
4. External LSA (Type 5): External LSA используются для анонсирования маршрутов, внешних к OSPF-автономной системе (например, маршруты, полученные от другого протокола маршрутизации, такого как BGP). В данном случае External LSA не используются.
