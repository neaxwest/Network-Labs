Задание 3
По топологии из Задания 1, сеть 172.16.1.0/24 анонсируется в OSPF через R4 и R5.

Какой путь или пути ( R1-R5, R1-R2-R4, R1-R3-R4) выберет R1 для данного маршрута? Почему? Соседства между R4-R5 нет.
Какие метрики для данного маршрута будет у R1 для данного маршрута через каждый путь? Конфигурация по умолчанию.
Какие изменения конфигурации необходимо сделать, чтоб правильно учитывалась пропускная способность каждого линка на этих путях?
После того, как будет учитываться правильно каждый линк, изменится ли ситуация с выбором маршрута на R1? Почему?
Приведите ответы на вопросы в свободной форме. В вопросе 3, напишите точно какую/какие команды необходимо использовать на маршрутизаторах


Решение:

1. Какой путь выберет R1?

R1 выберет R1-R5, потому что это самый короткий путь в той же области. R1-R2-R4 будет запасным.

2. Метрики?

R1-R5: Cost = 1 (по умолчанию)
R1-R2-R4: Cost = 2 (по умолчанию)

3. Что сделать, чтобы правильно учитывать пропускную способность?

На всех маршрутизаторах:

router ospf 1
auto-cost reference-bandwidth 10000

Это изменит формулу расчета метрики.

4. Изменится ли выбор маршрута?

Нет. R1-R5 всё равно останется лучше, потому что он в той же области. Теперь метрики будут другие, но принцип выбора не изменится:

•  R1-R5: Cost = 100
•  R1-R2-R4: Cost = 200

R1 выберет путь с наименьшей метрикой: R1-R5
