Задание 4. Определение DR и BDR
Сделайте предположение, какие маршрутизаторы будут выбраны в качестве DR и BDR. 
Узнайте, какие маршрутизаторы выбраны DR и BDR в реальности при помощи команд на маршрутизаторе. 
Какую команду необходимо использовать для получения информации о DR/BDR? Сошлись ли предположения с реальностью? Если нет, то почему?

1. Теоретический ответ:

Основываясь на предоставленных данных и правилах выбора DR/BDR в OSPF:

Правила выбора DR/BDR:
Высший OSPF Priority: Маршрутизатор с наивысшим приоритетом становится DR.
Высший Router ID: Если приоритет OSPF не настроен, то маршрутизатор с наивысшим Router ID становится DR.
Router ID: IP-адрес Loopback-интерфейса (если настроен). Иначе – наивысший IP-адрес на активном интерфейсе.
Non-Preemptive: DR и BDR остаются, пока не выйдут из строя.

Router ID (как указано в предыдущем сообщении):
R1: 1.1.1.1
R2: 2.2.2.2
R3: 3.3.3.3
R4: 192.168.40.4

Предполагаемые DR и BDR (теоретически):
DR: R4 (192.168.40.4), так как у него самый высокий Router ID.
BDR: R3 (3.3.3.3), как второй по величине Router ID.

2. Практика: Определение DR и BDR в реальности

Для определения DR и BDR команда:
show ip ospf neighbor

Router1#show ip ospf neighbor
Neighbor ID     Pri   State           Dead Time   Address         Interface
2.2.2.2           1   2WAY/DROTHER    00:00:40    192.168.1.2     FastEthernet0/0
3.3.3.3           1   FULL/BDR        00:00:34    192.168.1.3     FastEthernet0/0
192.168.40.4      1   FULL/DR         00:00:39    192.168.1.4     FastEthernet0/0

Router2#show ip ospf neighbor 
Neighbor ID     Pri   State           Dead Time   Address         Interface
1.1.1.1           1   2WAY/DROTHER    00:00:30    192.168.1.1     FastEthernet0/0
3.3.3.3           1   FULL/BDR        00:00:34    192.168.1.3     FastEthernet0/0
192.168.40.4      1   FULL/DR         00:00:39    192.168.1.4     FastEthernet0/0

Router3#show ip ospf neighbor
Neighbor ID     Pri   State           Dead Time   Address         Interface
2.2.2.2           1   FULL/DROTHER    00:00:40    192.168.1.2     FastEthernet0/0
1.1.1.1           1   FULL/DROTHER    00:00:30    192.168.1.1     FastEthernet0/0
192.168.40.4      1   FULL/DR         00:00:39    192.168.1.4     FastEthernet0/0

Router4#show ip ospf neighbor 
Neighbor ID     Pri   State           Dead Time   Address         Interface
2.2.2.2           1   FULL/DROTHER    00:00:40    192.168.1.2     FastEthernet0/0
1.1.1.1           1   FULL/DROTHER    00:00:30    192.168.1.1     FastEthernet0/0
3.3.3.3           1   FULL/BDR        00:00:34    192.168.1.3     FastEthernet0/0

Вывод команды:

R1: Видит R4 (192.168.40.4) как DR и R3 (3.3.3.3) как BDR.
R2: Видит R4 (192.168.40.4) как DR и R3 (3.3.3.3) как BDR.
R3: Видит R4 (192.168.40.4) как DR.
R4: Видит R3 (3.3.3.3) как BDR.


3. Реальные DR и BDR:
DR: R4 (192.168.40.4) – все маршрутизаторы согласны.
BDR: R3 (3.3.3.3) – R1, R2, R4 согласны, но сам R3 себя BDR не видит, хз почему.
